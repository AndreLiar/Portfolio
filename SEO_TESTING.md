# SEO Testing Guide for Portfolio

This document provides testing instructions for the implemented SEO enhancements.

## Testing Tools and URLs

### 1. Google Rich Results Test
- **URL**: https://search.google.com/test/rich-results
- **Test your live site**: Enter your portfolio URL
- **Test individual schemas**: Copy JSON-LD markup from your page source

### 2. Schema.org Validator
- **URL**: https://validator.schema.org/
- **Method**: Enter URL or paste JSON-LD markup directly

### 3. Google PageSpeed Insights
- **URL**: https://pagespeed.web.dev/
- **Test**: Core Web Vitals and SEO scores

### 4. Open Graph Debugger
- **Facebook**: https://developers.facebook.com/tools/debug/
- **LinkedIn**: https://www.linkedin.com/post-inspector/
- **Twitter**: https://cards-dev.twitter.com/validator

## Implemented Schema.org Structured Data

### 1. Person Schema
```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Laurel Kanmegne",
  "jobTitle": "Fullstack Software Engineer",
  "description": "Professional summary...",
  "url": "https://yoursite.com",
  "sameAs": ["github.com/...", "linkedin.com/..."],
  "knowsAbout": ["TypeScript", "React", "Next.js", ...]
}
```

### 2. SoftwareApplication Schema (for each project)
```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "Project Name",
  "description": "Project description...",
  "creator": { "@type": "Person", "name": "Laurel Kanmegne" },
  "programmingLanguage": ["TypeScript", "React"]
}
```

### 3. CollectionPage Schema
```json
{
  "@context": "https://schema.org",
  "@type": "CollectionPage",
  "name": "Laurel Kanmegne - Portfolio",
  "mainEntity": {
    "@type": "ItemList",
    "numberOfItems": 6,
    "itemListElement": [...]
  }
}
```

### 4. BreadcrumbList Schema
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://yoursite.com"
    }
  ]
}
```

## Enhanced Metadata

### Open Graph Tags
- `og:type`: website
- `og:title`: Portfolio title
- `og:description`: Professional description
- `og:image`: Portfolio screenshot (1200x630)
- `og:url`: Canonical URL

### Twitter Card
- `twitter:card`: summary_large_image
- `twitter:title`: Portfolio title
- `twitter:description`: Professional description
- `twitter:image`: Portfolio screenshot

### Additional Meta Tags
- Canonical URLs for each language
- Alternate language links (hreflang)
- Keywords relevant to your expertise
- Author and creator information

## SEO Files

### Robots.txt (`/public/robots.txt`)
- Allows all crawlers
- Specifies sitemap location
- Optimized for multilingual content

### Sitemap (`/src/app/sitemap.ts`)
- Auto-generated by Next.js
- Includes all language variants
- Updates automatically with content changes

### Web App Manifest (`/src/app/manifest.ts`)
- PWA configuration
- App icons and display settings
- Improves mobile user experience

## Testing Checklist

### ✅ Structured Data Validation
- [ ] Person schema validates without errors
- [ ] Project schemas validate without errors
- [ ] CollectionPage schema validates without errors
- [ ] BreadcrumbList schema validates without errors

### ✅ Meta Tags Testing
- [ ] Open Graph tags render correctly on Facebook/LinkedIn
- [ ] Twitter Cards display properly
- [ ] All meta descriptions under 160 characters
- [ ] Titles optimized for search (50-60 characters)

### ✅ Technical SEO
- [ ] Robots.txt accessible at /robots.txt
- [ ] Sitemap accessible at /sitemap.xml
- [ ] Canonical URLs set correctly
- [ ] Hreflang tags for language versions
- [ ] No duplicate content issues

### ✅ Performance Testing
- [ ] Core Web Vitals scores (LCP, FID, CLS)
- [ ] Mobile-friendly test passes
- [ ] Page load speed under 3 seconds
- [ ] Images optimized and properly sized

### ✅ Accessibility & SEO
- [ ] Proper heading hierarchy (H1-H6)
- [ ] Alt text for all images
- [ ] Semantic HTML structure
- [ ] Skip-to-content link present

## Common Issues to Watch For

- **Missing alt attributes** on images
- **Duplicate meta descriptions** across pages
- **Broken internal links** affecting crawlability
- **Slow page load times** hurting rankings
- **Mobile usability issues** impacting mobile search

Remember: SEO improvements take time to show results (typically 3-6 months for significant impact).
